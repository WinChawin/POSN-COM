## ข้อ 1 — Playlist (โครงสร้างข้อมูล: Deque + Stack)

### โจทย์

คุณกำลังจัดการ Playlist เพลง โดยมีคำสั่ง 5 แบบ:

| คำสั่ง | ความหมาย |
|--------|----------|
| `ADD s` | เพิ่มเพลงชื่อ `s` ต่อท้าย playlist |
| `PLAY` | เล่นเพลงแรกสุดใน playlist (พิมพ์ชื่อเพลงออกมา) แล้ว **ลบออกถาวร** |
| `STATUS` | พิมพ์เพลงทุกเพลงที่อยู่ใน playlist ตอนนี้ |
| `REMOVE k` | ลบ `k` เพลง**สุดท้าย**ออกจาก playlist (เก็บไว้สำหรับ restore ได้) |
| `RESTORE k` | เพิ่ม `k` เพลงที่ถูกลบล่าสุด (จาก REMOVE) กลับเข้าท้าย playlist |

### Input

- บรรทัดแรก: จำนวนคำสั่ง `Q`
- `Q` บรรทัดถัดมา: แต่ละบรรทัดเป็นคำสั่งตามรูปแบบข้างต้น

### Output

- สำหรับคำสั่ง `PLAY`: พิมพ์ชื่อเพลงที่เล่น 1 บรรทัด
- สำหรับคำสั่ง `STATUS`: พิมพ์เพลงทั้งหมดใน playlist (เพลงละบรรทัด)

### ตัวอย่าง

```
Input:
10
ADD songA
ADD songB
ADD songC
STATUS
PLAY
REMOVE 1
STATUS
RESTORE 1
STATUS
PLAY

Output:
songA
songB
songC
songA
songB
songB
songC
songB
```

### แนวคิด

- ใช้ **deque** เป็น playlist (เพิ่มท้าย, ลบหน้า, ลบท้าย)
- ใช้ **stack** เก็บเพลงที่ถูก REMOVE ไว้ เพื่อรองรับ RESTORE

### Complexity

- `ADD`, `PLAY`, `REMOVE`, `RESTORE`: O(1) ต่อเพลง (amortized)
- `STATUS`: O(n) ต่อจำนวนเพลงใน playlist

## ข้อ 2 — Copy (String Matching)

### โจทย์

ให้ string `s` มา จากนั้นมี `q` query แต่ละ query ให้ string `t` มา
ให้ตรวจสอบว่า `t` เป็น **substring** ของ `s` หรือไม่

### Input

- บรรทัดแรก: จำนวน `n` (ความยาวของ string)
- บรรทัดที่สอง: string `s`
- บรรทัดที่สาม: จำนวน query `q`
- `q` บรรทัดถัดมา: แต่ละบรรทัดมี `n_i` (ความยาว) แล้วตามด้วย string `t_i`

### Output

- สำหรับแต่ละ query: พิมพ์ `true` ถ้า `t` เป็น substring ของ `s` หรือ `false` ถ้าไม่ใช่

### ตัวอย่าง

```
Input:
6
sofsiof
2
1
s
3
sio

Output:
true
true
```

### แนวคิด

- ใช้ `s.find(t) != string::npos` ตรวจสอบ substring
- หรือใช้ KMP / Z-algorithm ถ้าต้องการประสิทธิภาพสูง

### Complexity

- Brute force (`s.find()`): O(n × m) ต่อ query
- KMP: O(n + m) ต่อ query

## ข้อ 3 — Generator (Prefix Sum + Brute Force)

### โจทย์

มีเครื่องกำเนิดไฟฟ้า `n` เครื่อง เรียงเป็นแถว แต่ละเครื่องมีกำลังการผลิต (จำนวนเต็ม)

ต้องการ **แบ่งเครื่องออกเป็น 2 กลุ่ม** โดยตัดที่ตำแหน่งใดตำแหน่งหนึ่ง (กลุ่มซ้าย: เครื่อง 1..i, กลุ่มขวา: เครื่อง i+1..n)

หาตำแหน่งที่ตัดแล้ว **ค่าเฉลี่ยของ 2 ฝั่งใกล้เคียงกันมากที่สุด** (ผลต่างสัมบูรณ์ของค่าเฉลี่ยทั้ง 2 ฝั่งน้อยที่สุด)

ถ้ามีหลายตำแหน่งที่ผลต่างเท่ากัน ให้พิมพ์ทุกตำแหน่ง

### Input

- บรรทัดแรก: จำนวนเครื่อง `n`
- บรรทัดที่สอง: กำลังการผลิตของแต่ละเครื่อง `a[0], a[1], ..., a[n-1]`

### Output

- พิมพ์ตำแหน่ง (0-indexed) ที่สามารถตัดได้ ทุกตำแหน่งที่ให้ผลต่างค่าเฉลี่ยน้อยที่สุด

### ตัวอย่าง

```
Input:
4
10 20 30 40

Output:
1
```

**คำอธิบาย:**
- ตัดที่ 0: ซ้าย = {10}, ขวา = {20,30,40} → avg = 10 vs 30 → diff = 20
- ตัดที่ 1: ซ้าย = {10,20}, ขวา = {30,40} → avg = 15 vs 35 → diff = 20
- ตัดที่ 2: ซ้าย = {10,20,30}, ขวา = {40} → avg = 20 vs 40 → diff = 20
- (ในตัวอย่างนี้เท่ากันหมด จริง ๆ ต้องดูตัวเลขจริงของโจทย์)

### แนวคิด

1. คำนวณ `sum` ทั้งหมด (prefix sum)
2. วน `i` จาก `0` ถึง `n-2`:
   - ค่าเฉลี่ยซ้าย = `prefix[i+1] / (i+1)`
   - ค่าเฉลี่ยขวา = `(sum - prefix[i+1]) / (n - i - 1)`
   - คำนวณ `diff = |avg_left - avg_right|` (ใช้ `double`)
3. เทียบกับค่า `mini`:
   - ถ้า `diff < mini` → ล้าง `ans`, เพิ่ม `i` เข้า `ans`, อัพเดท `mini`
   - ถ้า `diff == mini` → เพิ่ม `i` เข้า `ans`
4. พิมพ์ทุกตำแหน่งใน `ans`

### Complexity

- O(n)

## ข้อ 4 — Triclique (Graph: Brute Force Adjacency Matrix)

### โจทย์

ให้กราฟไม่มีทิศทาง (undirected graph) มี `n` โหนดและ `m` เส้นเชื่อม

**Triclique** (หรือ Triangle / Clique ขนาด 3) คือกลุ่ม 3 โหนดที่ทุกคู่เชื่อมกันหมด:
- `i — j` มี edge
- `j — k` มี edge
- `i — k` มี edge

ให้หาจำนวน triclique ทั้งหมดในกราฟ (หรือแสดงรายการ triclique ทั้งหมด)

### Input

- บรรทัดแรก: `n m` (จำนวนโหนด, จำนวน edge)
- `m` บรรทัดถัดมา: `u v` (edge ระหว่าง u กับ v)

### Output

- จำนวน triclique ทั้งหมด (หรือรายการ triclique แต่ละกลุ่ม ขึ้นอยู่กับโจทย์จริง)

### ตัวอย่าง

```
Input:
4 5
0 1
1 2
2 0
0 3
1 3

Output:
2
```

**คำอธิบาย:**
- Triclique 1: {0, 1, 2}
- Triclique 2: {0, 1, 3}

### แนวคิด

- เก็บกราฟเป็น **adjacency matrix** `adj[i][j]`
- วน 3 ชั้น: `i`, `j = i+1`, `k = j+1`
- ตรวจว่า `adj[i][j] && adj[j][k] && adj[i][k]`
- ถ้าใช่ → นับเป็น 1 triclique

```cpp
int cnt = 0;
for(int i = 0; i < n; i++)
    for(int j = i+1; j < n; j++)
        for(int k = j+1; k < n; k++)
            if(adj[i][j] && adj[j][k] && adj[i][k])
                cnt++;
```

### Complexity

- O(n³)
- ใช้ได้เมื่อ `n` ≤ ~500

## ข้อ 5 — โพรงกระรอก (BFS บน Grid พร้อมเงื่อนไขเวลา)

### โจทย์

มี grid ขนาด `n × m` แต่ละช่องมีตัวเลข `grid[i][j]` ซึ่งหมายถึง: **ถ้าจะเข้าช่องนี้ ต้องรอจนเวลาถึง `grid[i][j]` นาที ถึงจะเข้าได้**

กระรอกเริ่มต้นที่ช่อง `(0, 0)` ในเวลา `0` (สมมติ `grid[0][0] = 0`)
ต้องการวิ่งไปที่ช่อง `(n-1, m-1)` ให้ใช้เวลาน้อยที่สุด

การเคลื่อนที่แต่ละครั้งใช้เวลา 1 นาที แต่ถ้าเข้าช่องที่มีเลข `grid[i][j]` สูงกว่าเวลาปัจจุบัน ต้องรอจนถึงเวลานั้น

### Input

- บรรทัดแรก: `n m`
- `n` บรรทัดถัดมา: แต่ละบรรทัดมี `m` ตัวเลข แสดงค่า `grid[i][j]`

### Output

- เวลาน้อยที่สุดที่กระรอกจะถึงช่อง `(n-1, m-1)`

### ตัวอย่าง

```
Input:
2 2
0 8
7 2

Output:
3
```

**คำอธิบาย:**
- เวลา 0: อยู่ที่ (0,0)
- เวลา 1: ย้ายไป (1,0) → max(0+1, 7) = 7 ต้องรอถึงเวลา 7
- ถ้าไปทาง (0,0) → (0,1): เวลา = max(0+1, 8) = 8 → แล้วไป (1,1): max(8+1, 2) = 9
- ถ้าไปทาง (0,0) → (1,0): เวลา = max(0+1, 7) = 7 → แล้วไป (1,1): max(7+1, 2) = 8
- ทางที่ดีสุด: เวลา = 3 (ขึ้นอยู่กับค่าจริงของ grid)

### แนวคิด

- ใช้ **BFS** (หรือ Dijkstra เพราะน้ำหนักไม่เท่ากัน) บน grid
- **State**: ตำแหน่ง (r, c) กับเวลาที่มาถึง
- เมื่อจะเข้าช่อง (nr, nc): `new_time = max(current_time + 1, grid[nr][nc])`
- เก็บ `dist[r][c]` = เวลาน้อยสุดที่มาถึงช่อง (r, c)
- ใช้ priority queue (Dijkstra) เพื่อดึง state ที่เวลาน้อยสุดก่อน

### Complexity

- O(n × m × log(n × m)) ถ้าใช้ Dijkstra
- O(n × m) ถ้าใช้ 0-1 BFS (ขึ้นอยู่กับโครงสร้างน้ำหนัก)

## ข้อ 6 — ฟอกเงิน (BFS บน Directed Graph)

### โจทย์

มี **directed graph** ของคน `n` คน (หมายเลข `0` ถึง `n-1`)

คนหมายเลข `0` คือ "บิ๊ก" (หัวหน้าองค์กรฟอกเงิน) ซึ่ง **ไม่มี edge ออก** (degree out = 0)

คนหมายเลข `i` ถือเป็น **"คนฟอกเงินรายย่อย"** ก็ต่อเมื่อ:
1. สามารถ **ส่งเงินไปถึงคนหมายเลข 0** ได้ (มี path จาก `i` ไป `0`)
2. คนหมายเลข 0 สามารถ **ส่งเงินกลับมาถึง `i`** ได้ (มี path จาก `0` ไป `i`)

> หมายเหตุ: เนื่องจากบิ๊ก (0) ไม่มี edge ออก เงื่อนไขข้อ 2 ต้องตีความจากกราฟต้นฉบับว่า มี path จาก `0` ไป `i` ในกราฟ **reverse** (หรือโจทย์อาจให้ edge เป็นแบบ bidirectional ในบางกรณี)

ให้หาว่ามีคนฟอกเงินรายย่อยกี่คน (หรือแสดงรายชื่อ)

### Input

- บรรทัดแรก: `n m` (จำนวนคน, จำนวน edge)
- `m` บรรทัดถัดมา: `u v` (มี edge จาก u ไป v)

### Output

- จำนวนคนฟอกเงินรายย่อย (หรือรายการหมายเลข)

### ตัวอย่าง

```
Input:
5 5
1 0
2 1
3 2
0 3
4 1

Output:
3
```

**คำอธิบาย:**
- คน 1: ส่งเงินไป 0 ได้ (1→0) และ 0 ส่งกลับมาได้ (0→3→2→1) ✓
- คน 2: ส่งเงินไป 0 ได้ (2→1→0) และ 0 ส่งกลับมาได้ (0→3→2) ✓
- คน 3: ส่งเงินไป 0 ได้ (3→2→1→0) และ 0 ส่งกลับมาได้ (0→3) ✓
- คน 4: ส่งเงินไป 0 ได้ (4→1→0) แต่ 0 ส่งกลับ 4 ไม่ได้ ✗

### แนวคิด

1. BFS/DFS จากทุก node `i` บนกราฟปกติ → สร้าง `can_reach_0[i]` (i ไปถึง 0 ได้ไหม)
2. BFS/DFS จาก node `0` บนกราฟ **reverse** → สร้าง `reachable_from_0[i]` (0 ไปถึง i ได้ไหม)
3. คนที่ `can_reach_0[i] && reachable_from_0[i]` คือคนฟอกเงินรายย่อย

**วิธีที่ดีกว่า:**
1. สร้างกราฟ **reverse** ของกราฟต้นฉบับ
2. BFS จาก `0` บนกราฟ reverse → ได้ set ของ node ที่ "ส่งเงินไป 0 ได้"
3. BFS จาก `0` บนกราฟปกติ → ได้ set ของ node ที่ "0 ส่งเงินไปถึงได้"
4. คำตอบ = intersection ของ 2 set (ไม่รวม 0 เอง)

### Complexity

- O(n + m)

## ข้อ 7 — ทางด่วน (Modified Dijkstra: Minimax Path)

### โจทย์

มี **directed graph** ที่มี `n` โหนดและ `m` เส้นเชื่อม แต่ละเส้นมีค่าผ่านทาง (weight)

ต้องการเดินทางจากโหนด `1` ไปยังโหนด `n`

**ค่าใช้จ่ายของเส้นทาง** = ค่า edge ที่แพงที่สุดตลอดเส้นทาง (bottleneck / minimax)

หาเส้นทางที่ทำให้ค่า edge แพงสุดมีค่าน้อยที่สุด

### Input

- บรรทัดแรก: `n m`
- `m` บรรทัดถัดมา: `u v w` (edge จาก u ไป v น้ำหนัก w)

### Output

- ค่า edge แพงที่สุดบนเส้นทางที่ดีที่สุด

### ตัวอย่าง

```
Input:
5 5
1 3 8
3 2 4
2 5 7
1 5 9
3 5 10

Output:
8
```

**คำอธิบาย:**
- เส้นทาง 1→3→2→5: edge ที่แพงสุด = max(8, 4, 7) = **8**
- เส้นทาง 1→5: edge ที่แพงสุด = max(9) = **9**
- เส้นทาง 1→3→5: edge ที่แพงสุด = max(8, 10) = **10**
- ทางที่ดีที่สุดคือ 1→3→2→5 ได้ค่า **8**

### แนวคิด

- ใช้ **Dijkstra ดัดแปลง**:
  - แทนที่จะรวม weight (`dist[v] = dist[u] + w`) → ใช้ max (`dist[v] = max(dist[u], w)`)
  - priority queue เรียงตาม dist น้อยสุด (min-heap)
- **State**: `dist[u]` = ค่า edge แพงสุดบนเส้นทางที่ดีที่สุดจากโหนด 1 ถึง u
- **Relaxation**: `if max(dist[u], w) < dist[v]` → update `dist[v] = max(dist[u], w)`

```cpp
// Dijkstra แบบ minimax
dist[1] = 0;
priority_queue<pii, vector<pii>, greater<pii>> pq;
pq.push({0, 1});
while(!pq.empty()){
    auto [d, u] = pq.top(); pq.pop();
    if(d > dist[u]) continue;
    for(auto [v, w] : adj[u]){
        int nd = max(dist[u], w);
        if(nd < dist[v]){
            dist[v] = nd;
            pq.push({nd, v});
        }
    }
}
cout << dist[n] << "\n";
```

### Complexity

- O((n + m) log n) — เท่ากับ Dijkstra ปกติ

## ข้อ 8 — Balloon (Burst Balloons DP)

### โจทย์

เหมือนโจทย์ **Leetcode 312 - Burst Balloons**

มีลูกโป่ง `n` ลูก เรียงเป็นแถว ลูกที่ `i` มีค่า `nums[i]`

เมื่อเจาะลูกที่ `i` จะได้คะแนน = `nums[left] × nums[i] × nums[right]`
(โดย `left` และ `right` คือลูกโป่งข้างเคียงที่ยังไม่ถูกเจาะ)

ถ้าไม่มีลูกข้าง ให้ถือว่ามีลูกโป่งจำลองค่า `1` อยู่ที่ขอบทั้ง 2 ด้าน

หาคะแนนรวมสูงสุดที่ทำได้จากการเจาะลูกโป่งทั้งหมด

### Input

- บรรทัดแรก: จำนวนลูกโป่ง `n` (n ≤ 200)
- บรรทัดที่สอง: ค่าของลูกโป่งแต่ละลูก `nums[0], nums[1], ..., nums[n-1]`

### Output

- คะแนนรวมสูงสุด

### ตัวอย่าง

```
Input:
4
3 1 5 8

Output:
167
```

**คำอธิบาย:**
- เจาะลูก 1 (ค่า 1): ได้ 3×1×5 = 15
- เจาะลูก 0 (ค่า 3): ได้ 1×3×5 = 15
- เจาะลูก 2 (ค่า 5): ได้ 1×5×8 = 40
- เจาะลูก 3 (ค่า 8): ได้ 1×8×1 = 8
- ไม่ใช่วิธีที่ดีที่สุด — ลำดับที่ดีที่สุดให้ 167

### แนวคิด

- **Interval DP**: `dp[l][r]` = คะแนนสูงสุดจากการเจาะลูกโป่งทั้งหมดใน range `(l, r)` (exclusive)
- เพิ่มลูกโป่งจำลองค่า 1 ที่ตำแหน่ง 0 และ n+1
- สำหรับแต่ละ `k` ใน `(l, r)`: ให้ `k` เป็นลูก**สุดท้าย**ที่เจาะ
- `dp[l][r] = max(dp[l][k] + dp[k][r] + nums[l] × nums[k] × nums[r])`

```cpp
// เพิ่ม boundary
vector<int> a(n+2);
a[0] = a[n+1] = 1;
for(int i = 1; i <= n; i++) a[i] = nums[i-1];

// dp
int dp[n+2][n+2] = {};
for(int len = 2; len <= n+1; len++){
    for(int l = 0; l+len <= n+1; l++){
        int r = l+len;
        for(int k = l+1; k < r; k++){
            dp[l][r] = max(dp[l][r], dp[l][k] + dp[k][r] + a[l]*a[k]*a[r]);
        }
    }
}
cout << dp[0][n+1] << "\n";
```

### Complexity

- O(n³)

## ข้อ 9 — Merge Book (BST + Level Order Traversal)

### โจทย์

มีหนังสือ 2 ชุด:
- **ชุด N**: มี `n` เล่ม แต่ละเล่มมีค่า (ตัวเลข)
- **ชุด M**: มี `m` เล่ม แต่ละเล่มมีค่า (ตัวเลข)

ขั้นตอน:
1. **สร้าง BST** จากชุด N โดยใส่ทีละเล่มตามลำดับที่ให้มา
2. **สร้าง BST** จากชุด M โดยใส่ทีละเล่มตามลำดับที่ให้มา
3. **พิมพ์** BST ของชุด N แบบ **level-order traversal**
4. **พิมพ์** BST ของชุด M แบบ **level-order traversal**
5. นำหนังสือจากชุด M มา **ใส่เข้า BST ของชุด N** ตามลำดับ level-order ของ BST M
6. **พิมพ์** BST ของชุด N (ที่ merge แล้ว) แบบ **level-order traversal**

### Input

- บรรทัดแรก: `n` (จำนวนหนังสือชุด N)
- บรรทัดที่สอง: ค่าหนังสือชุด N ทีละเล่ม
- บรรทัดที่สาม: `m` (จำนวนหนังสือชุด M)
- บรรทัดที่สี่: ค่าหนังสือชุด M ทีละเล่ม

### Output

- บรรทัดที่ 1: Level-order ของ BST N
- บรรทัดที่ 2: Level-order ของ BST M
- บรรทัดที่ 3: Level-order ของ BST N หลัง merge

### ตัวอย่าง

```
Input:
3
7 5 215
3
556 545 -85

Output:
7 5 215
556 545 -85
7 5 215 556 545 -85
```

> (Output อาจไม่ตรงกับข้อมูลจริง — ตัวอย่างนี้เป็นการประมาณจากโน้ต)

### แนวคิด

1. Implement **BST** พร้อม insert
2. Implement **level-order traversal** (BFS โดยใช้ queue)
3. ดึง level-order ของ BST M มาเป็นลำดับ
4. insert ค่าเหล่านั้นเข้า BST N ตามลำดับ
5. พิมพ์ level-order ของ BST N อีกรอบ

### Complexity

- BST Insert: O(h) ต่อครั้ง (worst case O(n) ถ้า degenerate)
- Level-order: O(n)
- รวม: O((n + m) × h)

## ข้อ 10 — หาคนงาน (Optimization: Sort + Greedy / Recursion)

### โจทย์

มีคนงาน `n` คน ต้องเลือก `k` คน

คนงานแต่ละคน `i` มี:
- ค่าคุณภาพ `q[i]`
- ค่าแรงขั้นต่ำ `w[i]`

เงื่อนไข: คนงานทุกคนที่เลือกจะได้รับเงินตาม **อัตราส่วนของคุณภาพ** คือ ถ้าอัตราค่าแรงต่อคุณภาพคือ `r` แล้วคนงาน `i` จะได้ `r × q[i]`

แต่ทุกคนต้องได้เงินไม่ต่ำกว่าค่าแรงขั้นต่ำ: `r × q[i] ≥ w[i]` → `r ≥ w[i] / q[i]`

ดังนั้น rate `r` = `max(w[i]/q[i])` ของคนที่เลือก

**ค่าใช้จ่ายรวม** = `r × sum(q[i] ของคนที่เลือก)` = `max(w[i]/q[i]) × sum(q[i])`

หาค่าใช้จ่ายรวมที่ **น้อยที่สุด** ในการจ้างคนงาน `k` คน

### Input

- บรรทัดแรก: `n k`
- `n` บรรทัดถัดมา: `q[i] w[i]` (คุณภาพ และ ค่าแรงขั้นต่ำ)

### Output

- ค่าใช้จ่ายรวมน้อยที่สุด (อาจเป็นทศนิยม)

### ตัวอย่าง

```
Input:
3 2
10 70
20 50
5 30

Output:105
```

**คำอธิบาย:**
- คำนวณ ratio `c[i] = w[i] / q[i]`:
  - คน 0: c = 70/10 = 7
  - คน 1: c = 50/20 = 2.5
  - คน 2: c = 30/5 = 6
- เลือกคน 0 + คน 2: max(c) = max(7, 6) = 7, sum(q) = 10+5 = 15, cost = 7×15 = **105**
- เลือกคน 0 + คน 1: max(c) = max(7, 2.5) = 7, sum(q) = 10+20 = 30, cost = 7×30 = 210
- เลือกคน 1 + คน 2: max(c) = max(2.5, 6) = 6, sum(q) = 20+5 = 25, cost = 6×25 = 150
- ค่าน้อยสุด = **105**

### แนวคิด

**วิธีจากโน้ต (Recursion/Brute Force):**
- State: `(maxi, sum, num, visited)`
  - `maxi` = max ของ ratio c ของคนที่เลือกแล้ว
  - `sum` = sum ของคุณภาพ q ของคนที่เลือกแล้ว
  - `num` = จำนวนคนที่เลือกแล้ว
  - `visited` = set ของคนที่เลือกแล้ว
- ถ้า `num == k` → `ans = min(ans, maxi × sum)`
- วนเลือกคนที่ยังไม่ได้เลือก แล้ว recurse

**วิธีที่ดีกว่า (Sort + Max-Heap) — เหมือน Leetcode 857:**
1. คำนวณ ratio `c[i] = w[i] / q[i]` สำหรับทุกคน
2. Sort คนงานตาม `c[i]` จากน้อยไปมาก
3. ใช้ max-heap เก็บ `q[i]` ของคนที่เลือก
4. วนจากคน ratio น้อยไปมาก:
   - เพิ่ม `q[i]` ลง heap, บวก sum
   - ถ้า heap มีมากกว่า `k` คน → เอาคนที่ `q` สูงสุดออก (ลด sum)
   - ถ้า heap มี `k` คนพอดี → `ans = min(ans, c[i] × sum)`

### Complexity

- Brute force: O(C(n,k)) — ใช้ได้เมื่อ n เล็ก
- Sort + Heap: O(n log n)
